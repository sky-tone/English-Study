## 2. 注册微信小程序账号

> 如果只是本地调试，可以使用**测试号**跳过此步骤（见第 4 步的「使用测试号」）。

### 2.1 注册流程

1. 打开 [微信公众平台](https://mp.weixin.qq.com/)
2. 点击右上角 **「立即注册」**
3. 选择账号类型 → **「小程序」**
4. 填写邮箱、密码，完成邮箱验证
5. 选择主体类型 → **「个人」**
6. 填写管理员信息（姓名、身份证号、手机号、微信扫码绑定）
7. 注册完成后进入小程序管理后台

### 2.2 获取 AppID

1. 登录 [微信公众平台](https://mp.weixin.qq.com/)
2. 左侧菜单 → **「开发管理」** → **「开发设置」**
3. 找到 **「AppID (小程序 ID)」**，复制保存
4. 格式类似：`wx1234567890abcdef`

## 3. 安装微信开发者工具

### 3.1 下载

前往官方下载页：[微信开发者工具下载](https://developers.weixin.qq.com/miniprogram/dev/devtools/download.html)

选择对应操作系统版本（推荐**稳定版 Stable Build**）。

### 3.2 安装与登录

1. 安装完成后打开微信开发者工具
2. 使用**微信扫码**登录（此微信需为小程序管理员或开发者）

## 4. 导入项目到开发者工具

### 4.1 克隆项目代码

```bash
git clone https://github.com/sky-tone/English-Study.git
cd English-Study
```

### 4.2 导入到微信开发者工具

1. 打开微信开发者工具
2. 选择 **「+」** 或 **「导入项目」**
3. 填写以下信息：

| 字段 | 值 |
|------|-----|
| **项目名称** | 搭搭乐 (或自定义) |
| **目录** | 选择克隆仓库中的 `miniprogram/` 文件夹 |
| **AppID** | 见下方说明 |
| **开发模式** | 小程序 |

> ⚠️ **重要**：目录必须选择 `miniprogram/` 子文件夹，不是项目根目录！

### 4.3 AppID 配置

**方式一：使用测试号（推荐快速体验）**

- 在导入时 AppID 栏点击 **「使用测试号」**
- 无需注册小程序账号即可在模拟器中运行
- **限制**：测试号无法进行真机预览和上传发布

**方式二：使用正式 AppID**

1. 将 `miniprogram/project.config.json` 中的 `appid` 字段替换为你的 AppID：

```json
{
  "appid": "wx你的真实AppID"
}
```

2. 在导入项目时填入同一个 AppID
3. 可以进行真机预览、体验版发布等全部功能

### 4.4 导入成功

导入成功后，你应该看到：
- 左侧**模拟器**中显示搭搭乐首页（角色选择页面）
- 编辑器中能看到 `pages/`、`components/`、`utils/` 等目录
- 控制台无报错（首次编译可能有警告，属正常现象）

## 5. 模拟器调试

### 5.1 编译与运行

1. 点击工具栏 **「编译」** 按钮（或快捷键 `Ctrl/Cmd + B`）
2. 模拟器中将显示小程序首页

### 5.2 调试工具

| 工具 | 用途 |
|------|------|
| **Console** | 查看 `console.log` 输出和错误信息 |
| **Network** | 查看网络请求（Phase 1 仅本地数据，无网络请求） |
| **AppData** | 查看页面数据和全局数据（globalData） |
| **Storage** | 查看本地存储（errorLog、radarData 等） |
| **WXML** | 查看页面元素结构，类似浏览器的 Elements 面板 |

### 5.3 模拟不同机型

- 在模拟器顶部的设备选择栏中切换不同机型（iPhone、Android 等）
- 建议测试：iPhone 6/7/8、iPhone X、iPhone 12/13、Android 主流机型

### 5.4 页面导航测试

项目包含 4 个页面，可在模拟器中依次访问：

| 页面路径 | 说明 |
|----------|------|
| `pages/index/index` | 🏠 首页 — 角色选择 & 关卡预览 |
| `pages/level/level` | 🎮 闯关页面 — 核心积木拼装玩法 |
| `pages/parent/parent` | 👩 家长中心 — 雷达图 & 错题分析 |
| `pages/teacher/teacher` | 👨‍🏫 教师管理端 — 建班 & 作业管理 |

可通过以下方式导航到特定页面：
1. 点击工具栏的 **「编译」** 旁边的下拉箭头
2. 选择 **「添加编译模式」**
3. 在「启动页面」中输入目标页面路径，如 `pages/level/level`
4. 可选择添加「启动参数」，如 `levelId=level_1`

## 6. 真机预览（推送到手机微信）

> ⚠️ 需要使用**正式 AppID**（测试号不支持真机预览）

### 6.1 预览操作

1. 在微信开发者工具的工具栏中点击 **「预览」** 按钮
2. 等待编译打包完成
3. 屏幕上将显示一个**二维码**
4. 用**手机微信** → 扫一扫 → 扫描此二维码
5. 小程序将在手机微信中打开运行

### 6.2 预览注意事项

- 预览码**有效期 30 分钟**，过期需重新生成
- 只有**项目管理员和开发者**的微信才能扫码预览
- 每次修改代码后需重新点击「预览」生成新的二维码
- 手机需处于**联网状态**

### 6.3 真机调试（高级）

如果需要在真机上查看 console 日志和调试信息：

1. 点击工具栏 **「真机调试」** 按钮（在「预览」旁边）
2. 扫码后，开发者工具会建立与手机的远程调试连接
3. 可在电脑端看到手机上的 console 输出、网络请求、Storage 数据
4. 支持断点调试

## 7. 体验版发布（分享给其他人测试）

> 适用于将小程序发送给团队成员或测试用户体验。

### 7.1 上传代码

1. 在微信开发者工具中点击 **「上传」** 按钮
2. 填写版本号（如 `1.0.0`）和项目备注
3. 上传成功后，代码会出现在微信公众平台后台

### 7.2 设置体验版

1. 登录 [微信公众平台](https://mp.weixin.qq.com/)
2. 左侧菜单 → **「管理」** → **「版本管理」**
3. 在「开发版本」中找到刚上传的版本
4. 点击该版本旁的 **「设为体验版」**
5. 系统会生成一个**体验版二维码**

### 7.3 添加体验者

1. 左侧菜单 → **「管理」** → **「成员管理」**
2. 在「体验成员」区域点击 **「添加」**
3. 输入测试人员的**微信号**
4. 对方确认后即可通过扫描体验版二维码使用小程序

### 7.4 体验版限制

- 最多添加 **15 个体验者**（个人主体小程序）
- 体验版有效期跟随上传的版本，更新代码需重新上传
- 体验版右上角会显示 **「体验版」** 绿色标签

## 10. 常见问题排查

### Q1: 导入项目后模拟器显示空白

**原因**：可能选错了项目目录。

**解决**：确保导入的目录是 `miniprogram/` 子文件夹（包含 `app.json` 的目录），而不是项目根目录。

---

### Q2: 提示 "appid 不合法"

**解决方案**：
- 方式一：使用测试号 — 在导入项目时选择「使用测试号」
- 方式二：替换 AppID — 将 `project.config.json` 中的 `"appid": "wxDEV_PLACEHOLDER"` 替换为你的真实 AppID

---

### Q3: 无法真机预览

**检查以下条件**：
1. 使用的是正式 AppID（非测试号）
2. 扫码的微信与小程序管理员/开发者是同一个微信
3. 如果是其他人测试，需先在公众平台添加为「体验成员」
4. 手机联网正常

---

### Q4: Canvas 雷达图不显示

**可能原因**：
- 开发者工具的 Canvas 渲染与真机有差异
- 尝试在**真机预览**中查看，Canvas 2D 在真机上表现更好
- 检查控制台是否有 Canvas 相关错误

---

### Q5: 单元测试报错 "Cannot find module"

```bash
# 确保在项目根目录下安装依赖
cd English-Study
npm install

# 然后再运行测试
npm test
```

---

### Q6: 如何清除本地测试数据

在微信开发者工具中：
1. 点击顶部工具栏 **「清缓存」** → **「清除全部」**
2. 或在调试器 → Storage 面板中手动删除 `errorLog`、`radarData` 等 key
