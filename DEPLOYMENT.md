## 🚀 快速路线：测试号 + 模拟器（5 分钟）

> 不需要注册任何账号，不需要改任何代码。

### 第一步：导入项目

1. 打开微信开发者工具
2. 在项目管理界面点击 **「+」**（或 **「导入项目」**）
3. 你会看到一个表单，按下面填写：

| 字段 | 填什么 |
|------|--------|
| **项目名称** | `搭搭乐` （随便填，自己认识就行） |
| **目录** | 点击「选择」→ 找到 `English-Study/miniprogram/` 文件夹 |
| **AppID** | 点击输入框下方的蓝色文字 **「使用测试号」** |

> [!CAUTION]
> ### 🚨 经典踩坑：目录选错
>
> **必须** 选择 `miniprogram/` 子文件夹，**不是** `English-Study/` 根目录。
>
> 怎么确认选对了？看你选的文件夹里有没有 `app.json` 文件。
>
> | 选择的目录 | 里面有什么 | 结果 |
> |-----------|-----------|------|
> | ✅ `English-Study/miniprogram/` | `app.json`, `app.js`, `pages/`... | 正确！ |
> | ❌ `English-Study/` | `package.json`, `DEPLOYMENT.md`... | 会报错 |
>
> 选错了会怎样？开发者工具会提示「不是一个有效的小程序项目」或者模拟器显示空白。

4. 点击 **「确定」** 导入

### 第二步：编译运行

1. 导入后，开发者工具自动编译，左侧**模拟器**区域会显示搭搭乐首页
2. 如果没自动编译，点击顶部工具栏的 **「编译」**（快捷键 `Ctrl+B` / `Cmd+B`）
3. 你应该看到：**蓝色导航栏**写着「搭搭乐」，下面有角色选择按钮和关卡列表

> 🎉 **到这里就能用了！** 在模拟器里点击各种按钮，体验积木拼装造句。

### 第三步：模拟器里玩一遍

1. **选择角色**：点击「学生」进入闯关，或点「家长」「教师」查看其他视角
2. **闯关体验**：
   - 点击关卡 1 进入
   - NPC Aki 会弹出对话，点击「我来帮你！Let's go! 🚀」关闭
   - 从下方积木库点击积木，组装句子
   - 故意选错（比如 I + are）看看错误提示
   - 拼好句子后点 Submit 提交
3. **切换机型**：模拟器顶部有设备下拉栏，建议切换 iPhone 6/7/8、iPhone X、Android 各看一遍

### 第四步：用 Compile Mode 快速跳转到指定页面

不想每次都从首页点进去？添加一个编译模式：

1. 点击工具栏 **「编译」** 按钮旁的 **下拉箭头 ▼**
2. 点击 **「添加编译模式」**
3. 填写：
   - **模式名称**：`闯关-关卡1`
   - **启动页面**：`pages/level/level`
   - **启动参数**：`levelId=level_1`
4. 点击「确定」→ 自动编译并直接进入闯关页面

常用编译模式参考：

| 模式名称 | 启动页面 | 启动参数 |
|----------|----------|----------|
| 闯关-关卡1 | `pages/level/level` | `levelId=level_1` |
| 闯关-关卡2 | `pages/level/level` | `levelId=level_2` |
| 闯关-关卡3 | `pages/level/level` | `levelId=level_3` |
| 家长中心 | `pages/parent/parent` | （留空） |
| 教师端 | `pages/teacher/teacher` | （留空） |

### 第五步：使用调试工具

开发者工具底部的 **Debugger 面板** 有以下标签页：

| 标签 | 用途 | 本项目的关注点 |
|------|------|---------------|
| **Console** | 查看日志和报错 | 碰撞引擎的拒绝消息会打印在这里 |
| **AppData** | 实时查看页面 data | 查看 `currentSentence`、`totalScore` 等状态 |
| **Storage** | 查看本地存储 | 闯关后查看 `errorLog` 和 `radarData` 数据 |
| **WXML** | 查看页面元素树 | 检查积木组件渲染、定位布局问题 |
| **Network** | 查看网络请求 | Phase 1 无网络请求，此面板应该是空的 |

## 📱 真机路线：正式 AppID + 手机微信

> 让小程序真正跑在你的手机微信里。需要额外 10 分钟注册小程序账号。

### 第一步：注册微信小程序账号（已有可跳过）

1. 电脑浏览器打开 [微信公众平台](https://mp.weixin.qq.com/)
2. 点击右上角 **「立即注册」**
3. 选择账号类型 → **「小程序」**（不是公众号、不是企业微信）
4. 用一个**未绑定过公众平台的邮箱**注册
5. 去邮箱查收验证邮件，点击链接验证
6. 选择主体类型 → **「个人」**
7. 填写管理员信息：
   - 真实姓名 + 身份证号
   - 手机号 → 接收验证码
   - 用管理员微信扫码确认绑定
8. 完成注册，自动进入小程序管理后台

### 第二步：获取你的 AppID

1. 登录 [微信公众平台](https://mp.weixin.qq.com/)（用刚注册的账号）
2. 左侧菜单 → **「开发管理」**
3. 点击上方 Tab → **「开发设置」**
4. 页面上方就能看到：
   ```
   AppID (小程序ID)    wx1a2b3c4d5e6f7890
                        [复制]
   ```
5. 点击 **「复制」**，把 AppID 保存下来

### 第三步：替换项目中的 AppID

打开 `miniprogram/project.config.json`，找到这一行：

```json
"appid": "wxDEV_PLACEHOLDER",
```

把 `wxDEV_PLACEHOLDER` 替换成你自己的 AppID：

```json
"appid": "wx1a2b3c4d5e6f7890",
```

> [!IMPORTANT]
> ### 关于 `wxDEV_PLACEHOLDER` 的说明
>
> 项目代码里用 `wxDEV_PLACEHOLDER` 作为占位符，表示「这里需要填入真实 AppID」。
> 这**不是**一个有效的 AppID。如果不替换，会遇到「appid 不合法」的错误。
>
> - 用测试号（快速路线）：不用改，导入时选「使用测试号」会自动覆盖
> - 用真机（本路线）：**必须替换成你的真实 AppID**

### 第四步：导入项目

1. 打开微信开发者工具
2. 点击 **「+」** 导入项目
3. 填写：

| 字段 | 填什么 |
|------|--------|
| **项目名称** | `搭搭乐` |
| **目录** | 选择 `English-Study/miniprogram/` 文件夹 |
| **AppID** | 填入你在第二步复制的 AppID |

> [!CAUTION]
> 再次提醒：**目录选 `miniprogram/`**，不是根目录！

4. 点击 **「确定」**
5. 等待编译，模拟器中应该出现搭搭乐首页

### 第五步：确认基础库版本

项目的 `project.config.json` 里指定了基础库版本 `3.3.4`：

```json
"libVersion": "3.3.4",
```

一般不需要手动改。如果编译时提示基础库版本问题：

1. 点击顶部菜单 **「设置」** → **「项目设置」**
2. 找到 **「调试基础库」**
3. 确认版本 ≥ `3.3.4`（选最新版即可）

### 第六步：预览到手机

1. 确认模拟器中能正常运行
2. 点击工具栏的 **「预览」** 按钮
3. 等待编译打包（通常 5–10 秒）
4. 屏幕弹出一个 **QR Code 二维码**
5. 拿起手机 → 打开**微信** → **扫一扫** → 对准这个二维码
6. 手机微信中自动打开搭搭乐小程序

> #### 手机上你会看到什么
>
> 1. **首页**：蓝色顶栏「搭搭乐」，三个角色选择按钮，底部关卡列表
> 2. 点击「学生」→ 关卡 1 → 进入积木拼装界面
> 3. NPC Aki 弹出对话：带有 👽 头像的气泡动画
> 4. 下方五种颜色的积木面板，点击积木添加到句子区
> 5. 积木拼装区在屏幕上方，带虚线边框
> 6. 拼错时积木抖动 + 手机震动反馈

### 第七步：真机调试（可选，排查问题时用）

如果手机上出了问题，需要看日志：

1. 点击工具栏的 **「真机调试」**（在「预览」按钮旁边）
2. 手机微信扫码
3. 电脑端会弹出一个远程调试面板：
   - **Console**：看到手机上 `console.log` 的输出
   - **Storage**：看到手机上的本地存储数据
   - **WXML**：实时查看手机上的页面元素
4. 支持在电脑端打断点调试手机上运行的代码

### 预览注意事项

| 注意项 | 说明 |
|--------|------|
| **有效期** | 预览二维码有效期约 30 分钟，过期需重新生成 |
| **谁能扫** | 只有小程序管理员和已添加的开发者/体验者 |
| **改了代码** | 每次改代码后需要重新点「预览」生成新二维码 |
| **手机网络** | 手机需联网（虽然 Phase 1 是纯本地数据，但微信框架需要联网初始化） |

- [ ] 进入关卡时 NPC 对话自动弹出（带半透明遮罩）

- [ ] 对话气泡有弹入动画（pop-in）

**碰撞校验引擎（核心测试）：**

- [ ] ✅ `I` + `am` → 接受，积木正常添加到句子区
- [ ] ❌ `I` + `are` → 拒绝，积木抖动 + 显示提示「I 的好帮手是 am 哦！」
- [ ] ❌ `I` + `is` → 拒绝
- [ ] ✅ `He` / `She` / `Ben` + `is` → 接受
- [ ] ✅ `We` / `They` / `My family` + `are` → 接受
- [ ] ✅ `will` + `travel to Beijing` → 接受（will + 动词原形）

## ❓ 常见问题排查

### Q1: 导入项目后模拟器白屏 / 提示「不是有效项目」

**原因**：导入目录选错了。选了 `English-Study/` 根目录，而不是 `English-Study/miniprogram/`。

**解决**：
1. 在开发者工具中删除当前项目
2. 重新导入，目录选择 `miniprogram/` 文件夹
3. 验证方法：你选的文件夹里应该直接有 `app.json` 文件

---

### Q2: 提示「appid 不合法」或「AppID 与后台不匹配」

**原因**：项目默认的 `"appid": "wxDEV_PLACEHOLDER"` 是占位符，不是有效 AppID。

**解决（二选一）**：
- **快速路线**：导入项目时不填 AppID，点击 **「使用测试号」**
- **真机路线**：编辑 `miniprogram/project.config.json`，把 `wxDEV_PLACEHOLDER` 换成你的真实 AppID

---

### Q3: 点击「预览」报错，无法真机预览

**逐条排查**：

| 检查项 | 如何确认 |
|--------|---------|
| 是否使用了正式 AppID | 测试号不支持预览。打开 `project.config.json` 确认 `appid` 不是 placeholder |
| 微信是否是管理员 | 扫码的微信必须是注册小程序时绑定的管理员微信 |
| 开发者工具是否登录 | 工具右上角应显示你的微信头像，不是「未登录」 |
| 手机是否联网 | 虽然是本地数据，但微信框架初始化需要网络 |

---

### Q4: 模拟器里 Canvas 雷达图不显示 / 显示异常

**这是常见问题**。开发者工具对 Canvas 2D 的渲染与真机有差异。

**解决**：
1. 先在真机预览上查看 — Canvas 2D 在真机上表现通常更好
2. 检查 Console 面板是否有 Canvas 相关报错
3. 确认基础库版本 ≥ `3.3.4`（Canvas 2D API 需要较新版本）
4. 如果真机也不显示，尝试切换基础库到最新版本

---

### Q5: 积木点击无反应 / 颜色不对

**排查步骤**：
1. Console 面板是否有 JS 报错
2. 在 AppData 面板检查 `availableBlocks` 数据是否正确加载
3. 在 WXML 面板检查 block 组件是否渲染
4. 尝试「清缓存 → 全部清除」后重新编译

---

### Q6: `npm test` 报错 `Cannot find module` 或 `command not found`

```bash
# 确认你在项目根目录，不是 miniprogram/ 目录
pwd
# 应该输出：.../English-Study

# 安装依赖
npm install

# 再跑测试
npm test
```

如果 `node` 命令不存在，说明没装 Node.js。去 [nodejs.org](https://nodejs.org/) 下载安装 v16+。

---

### Q7: 如何清除所有测试数据（重置到初始状态）

在微信开发者工具中：

**方法一（推荐）**：工具栏 → **「清缓存」** → **「清除全部」**

**方法二（精确删除）**：
1. 打开调试器 → **Storage** 面板
2. 右键删除以下 key：
   - `errorLog` — 错题记录
   - `radarData` — 五维雷达数据
   - `role` — 角色选择
   - `studentId` — 学生 ID

---

### Q8: 编译警告 `sitemap 索引未配置` / `lazyCodeLoading` 提示

这些是微信开发者工具的**信息性提示**，不影响运行。项目已正确配置：

- `sitemap.json`：允许索引 index 和 level 页面，禁止索引 parent 和 teacher 页面
- `app.json` 中 `"lazyCodeLoading": "requiredComponents"` 已启用懒加载

可以安全忽略这些警告。

---

### Q9: 本项目有没有网络请求 / 需要配置服务器域名吗？

**不需要**。当前是 Phase 1 MVP，所有数据都使用 `mock-data.js` 提供的本地写死数据，没有任何 `wx.request` 调用。所以：

- 不需要配置服务器域名
- 不需要 HTTPS 证书
- 不需要后端服务
- Network 面板应该是空的

Phase 2 接入后端后才需要配置域名白名单。
